About Functioning of Django Web Application

The web application waits for HTTP requests from the web browser. When a request is received the application works out what is needed based on the URL and information in GET data. It reads information from a page_data dictonary. The application then returns a response to the web browser, dynamically creating an HTML page for the browser to display by inserting the retrieved data into placeholders in an HTML template.

The web application has 3 major components:

  1. View (views.py)

    A view is a request handler function, which receives HTTP requests and returns HTTP responses. Views access the data needed to satisfy requests via models, and delegate the formatting of the response to templates.
    Created different view functions to handle each different resource.

    • home view function: For chessboard page.
    • history view function : For chessboard history page
    • piecessummary view function: For chess piece summary page
    • about view function: For this current page

  2. URLS (urls.py)

    A URL mapper is used to redirect HTTP requests to the appropriate view based on the request URL.

  3. Templates (html file)

    A template is a text file defining the structure or layout of a file (such as an HTML page), with placeholders used to represent actual content. A view dynamically creates an HTML page using an HTML template, populating it with data from a model.
